\begin{abstract}
We rebuilt a generated robotics idea into a concrete data-engine study: should a robot data engine acquire new rollouts by uncovered physical failure mechanisms rather than by task labels, state diversity, uncertainty, or active failure prediction? The v4 rebuild implements a MuJoCo tabletop manipulation benchmark with a pusher, movable block, fixture, walls, pocket contact, friction/mass shifts, actuator limits, sensor dropout, physics-derived failure labels, iterative acquisition rounds, seven random seeds, uncertainty intervals, paired statistics, ablations, stress sweeps, and negative cases. The result is negative. On the decisive \texttt{combined\_tail\_stress} split, the proposed \texttt{failure\_stratified\_engine} reaches $0.643 \pm 0.058$ robust selector success, while the strongest non-oracle baseline, \texttt{failure\_prediction\_active\_learning}, reaches $0.675 \pm 0.055$. The paired success difference is $-0.032 \pm 0.047$. Failure stratification improves rare failure recall, but loses macro failure F1 and does not improve downstream robust selection. We therefore mark the paper \textbf{KILL/ARCHIVE} for ICLR main.
\end{abstract}

\section{Decision}
This document is the v4 ICLR-main gate for Paper 74, \emph{Data Engine Failure Stratification}. The previous version was killed because it only contained a synthetic template. The current rebuild adds a real local MuJoCo benchmark and implemented acquisition baselines. The rejection reason has changed: evidence now exists, but it does not support the central submission claim.

\section{Hypothesis}
The tested hypothesis is that selecting new robot data by uncovered failure mechanisms improves tail-robust action selection more than task-label balancing, state-diversity coresets, uncertainty sampling, and active failure prediction. The proposed method only passes the submission gate if mechanism stratification improves the downstream robust selector on the hardest combined-tail split while preserving failure macro F1, rare failure recall, and safety.

\section{Benchmark}
The benchmark uses MuJoCo dynamics for a planar pusher, a sliding block, top/bottom walls, a target pocket, and a fixed obstacle. Rollouts vary friction, mass, actuator authority, fixture placement, sensor dropout, and contact noise. Each rollout records task success, safety, progress, contact traces, acquisition features, and eight physics-derived failure labels: slip, jam, fixture collision, wall collision, actuator saturation, missed contact, sensor dropout, and timeout.

Five evaluation splits are used: nominal task balance, rare slip failures, jammed fixture failures, actuator-limit failures, and combined tail stress. The main evaluation contains 15,120 rollout-pool rows, 3,780 held-out rollout rows, 1,225 per-round metric rows, and 245 seed-level metric rows. The ablation study contains 210 acquisition-round rows, and the stress sweep contains 168 raw rows. All intervals are 95 percent confidence intervals over seed-level means.

\section{Methods}
The implemented acquisition methods are:
\begin{itemize}
\item \texttt{random\_sampling}: uniformly acquires from the unlabeled rollout pool.
\item \texttt{task\_label\_stratification}: balances acquisition across task/split labels.
\item \texttt{state\_diversity\_coreset}: selects diverse pre-rollout state and environment features.
\item \texttt{uncertainty\_sampling}: selects examples with high learned failure uncertainty.
\item \texttt{failure\_prediction\_active\_learning}: selects examples predicted likely to fail.
\item \texttt{failure\_stratified\_engine}: proposed method; balances acquisition across inferred and rare failure strata.
\item \texttt{oracle\_failure\_strata}: upper bound with access to true failure mechanism labels for selection.
\end{itemize}

\section{Stress Sweep and Negative Cases}
The stress sweep reinforces the same conclusion. At mild stress, all methods solve the task; at high stress, robust success collapses for all methods. At stress level 1.00, \texttt{failure\_stratified\_engine} and \texttt{failure\_prediction\_active\_learning} both reach $0.014 \pm 0.028$ robust success. The proposed method has higher rare recall at that stress level, but the selected policies still fail almost always.

\begin{figure}[h]
\centering
\includegraphics[width=0.82\linewidth]{../figures/failure_engine_stress_sweep.png}
\caption{Stress sweep success. Failure stratification does not maintain downstream success under severe combined-tail stress.}
\end{figure}

Negative cases concentrate around fixture-induced jams and missed-contact rollouts. The failure-stratified selector often identifies rare mechanisms, but the learned robust selector still chooses policies that collide with the fixture or miss contact under dropout. This is the core failure mode: mechanism coverage is not enough when the action selector cannot convert that coverage into a safer policy choice.

\section{Related Work Pressure}
The hostile prior-work set is crowded around failure detection, failure-reasoning data, runtime failure prediction, uncertainty-driven acquisition, and failure-mode analysis \citep{prior1,prior2,prior3,prior7}. In that context, failure-stratified data acquisition needs a clear downstream win. The present evidence does not provide one.

\section{Limitations}
This rebuild is not a real robot paper. It lacks hardware experiments, videos, external benchmark validation, and large learned robot-policy checkpoints. The local MuJoCo benchmark is useful for falsification, but not a polished public benchmark contribution. These limitations would matter even if the method won; because it loses the main downstream metric, they reinforce the archive decision.

\section{Conclusion}
Paper 74 should not be submitted to ICLR main. The v4 rebuild provides useful evidence, but the evidence is negative: failure-stratified acquisition improves some rare-label metrics without improving tail-robust robot data selection. The correct terminal state is \textbf{KILL/ARCHIVE}.
