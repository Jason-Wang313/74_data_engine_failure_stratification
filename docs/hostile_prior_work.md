# Hostile Prior Work

The local hostile prior-work slice puts pressure on the Paper 74 claim from four directions:

- Failure prediction can be used directly as an acquisition objective.
- Uncertainty sampling is a simple and strong data-engine baseline.
- Task and state diversity baselines are cheap and hard to beat.
- Failure-mode analysis and failure-reasoning datasets make the phrase "failure stratification" insufficiently novel without a downstream robotics win.

The v4 result does not clear this pressure. Active failure prediction is the strongest non-oracle baseline on the decisive split, and the proposed failure-stratified engine does not beat it.
